Ответы на вопросы: 1. Технологии, изученные мною в этом разделе учебника, предназначены для упращения и улучшения работы пользователя на эл. ресурсе. Например, пользователь долго работал над проектом и случайно нажал на кнопку выхода и не сохранился, в целях избежания ситуаций с несохранёнными данными, мы можем использовать событие beforeunload. С помощью атрибут defer можно указать браузеру на продолжение загрузки страницы даже в фоновом режиме, DOMContentLoaded сработает только после полной загрузки.. Атрибут asyns позволит сделать скрипт независимым, DOMContentLoaded сработает и до полной загрузки. Так же можно отслеживать загрузку абсолютно любых ресурсов, load - подскажет пользователю, что загрузка произведена успешно, error - наоборот, укажет на то, что произошла ошибка. 2. При создании ресурса с онлайн играми я могу использовать DOMContentLoaded для того, чтобы после полной загрузки страницы браузер мог попытаться заполнить логин и пароль за пользователя (если разрешено), событие beforeunload я смогу использовать для того, чтобы пользователь смог подтвердить свой уход с сайта, т.к. возможен случайный выход и всё несохранённые данные исчезнут. 3. На сайте :https://e.sfu-kras.ru/login/index.php после полной прогрузки страницы, вводятся сохранённые логин и пароль.
